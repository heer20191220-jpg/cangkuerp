# 仓库 ERP 联网版规划

## 目标

把当前商品录入工具升级为公司员工可联网使用的仓库 ERP。第一阶段先做“商品资料中心”，支持多人登录、统一保存、权限控制、商品资料维护、海关编码匹配和导出。

## 当前试运行版定位

当前版本已经具备基础后端服务和统一数据文件，适合公司内网小范围试运行：

- 员工通过浏览器访问同一台服务器
- 商品数据统一保存到 `data/db.json`
- 支持管理员、录入员、查看员三类角色
- 支持商品新增、编辑、删除、搜索、导出
- 支持操作日志

当前版本不建议直接作为长期正式系统使用，因为数据仍保存在 JSON 文件中，不是正式数据库。

## 一期上线范围

一期先做“商品资料中心”，不要一开始就把采购、入库、出库、财务全部放进去。

必须功能：

- 员工账号登录
- 角色权限：管理员、录入员、查看员
- 商品新增、编辑、查询、删除
- 字段：店铺名、自定义商品名、中文品名、英文品名、中文材质、英文材质、中文用途、英文用途、产品类别、长宽高、重量、采购价、体积、计费重、海关编码
- 产品类别下拉：普货、纺织、带电、带磁、木质、敏感
- 根据中文品名查询海关编码
- 商品列表筛选和搜索
- CSV/Excel 导入导出
- 操作日志：谁在什么时候新增或修改了什么
- 每日数据备份

## 二期范围

二期再扩展仓库业务：

- SKU 编码规则
- 供应商资料
- 采购单
- 入库单
- 库存数量
- 出库单
- 库存流水
- 图片附件
- 批量修改商品资料

## 三期范围

三期再做管理分析：

- 库存预警
- 成本统计
- 店铺维度报表
- 员工工作量统计
- 海关编码维护后台
- 数据备份和恢复

## 推荐正式架构

正式联网版建议拆成三层：

- 前端：员工在浏览器里使用的 ERP 页面
- 后端：处理登录、权限、商品保存、编码查询
- 数据库：统一保存所有公司商品资料

推荐技术路线：

- 前端：现有 HTML/CSS/JS 可继续演进，后续再迁移 React 或 Vue
- 后端：Node.js
- 数据库：PostgreSQL 或 SQLite
- 部署：公司服务器或云服务器
- 文件导入导出：Excel/CSV

## 近期决策

下一步需要确认：

- 公司员工大概多少人同时使用
- 是否只在公司内网使用，还是外网也要访问
- 是否需要手机端录入
- 海关编码是否已有现成表格
- 商品数据是否需要从旧表格导入
