## 当前定位

项目当前处于“本地/公司内网试运行版”阶段。

当前系统不是纯静态网页，已经有 Node.js 后端服务。使用方式是：在一台电脑上启动服务，员工通过浏览器访问 `http://localhost:3000` 或内网地址 `http://服务器IP:3000` 使用。

数据目前保存在本地 JSON 文件：

`data/db.json`

这适合小范围试运行和业务流程验证。后续如果要多人长期使用，建议迁移到 SQLite 或 PostgreSQL。

## 当前限制

- 数据仍保存在 JSON 文件里，不适合长期高并发。
- 还没有员工账号管理页面。
- 还没有海关编码词库维护页面。
- 角色权限有基础后端判断，但前端按钮级权限还需要继续细化。
- CSV 导入已具备，但还不是完整 Excel 工作流。
- 1688 采集依赖目标网页可访问性，遇到反爬或登录页时可能失败。
- Mercado Libre 已按 OAuth 流程接入，支持生成授权链接、回调换 token、自动绑定 Seller ID。
- noon 和 takealot 已预留 API Key 适配器，目前测试连接为密钥格式占位校验。
- 店铺订单同步、商品同步、库存同步方法已在适配器接口中预留，业务接口尚未开放。
- 还没有正式部署脚本、系统服务配置和自动每日备份任务。
- 还没有完整测试脚本。
