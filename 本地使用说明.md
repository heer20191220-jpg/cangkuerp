# 仓库 ERP 本地使用说明

## 启动

双击：

`启动联网ERP.bat`

启动后在浏览器打开：

`http://localhost:3000`

同一公司内网的其他电脑可以访问：

`http://服务器IP:3000`

服务器 IP 可以在启动这台电脑上运行 `ipconfig` 查看，一般是 `192.168.x.x` 或 `10.x.x.x`。

## 默认账号

管理员：

- 账号：`admin`
- 密码：`admin123`

录入员：

- 账号：`luru`
- 密码：`123456`

正式给公司使用前，请先修改默认密码：

```powershell
node set-password.js admin 新管理员密码
node set-password.js luru 新录入员密码
```

密码至少 8 位。修改后，旧登录状态会自动失效。

## 美客多店铺授权

1. 登录 Mercado Libre 开发者后台，创建应用。
2. 在应用里设置 Redirect URI：

`http://localhost:3000/oauth/mercadolibre/callback`

如果是局域网其他电脑访问，把 `localhost` 换成启动 ERP 的电脑 IP，例如：

`http://192.168.1.23:3000/oauth/mercadolibre/callback`

3. 在 ERP 里进入“店铺管理”。
4. 点击“新增店铺”。
5. 平台选择 `Mercado Libre / 美客多`。
6. 授权方式保持 `OAuth`。
7. 填写店铺名称、Client ID/API Key、Client Secret/API Secret。
8. 点击“开始美客多授权”。
9. 在打开的美客多页面登录并授权。
10. 授权成功后会自动回到 ERP，并自动保存 access token、refresh token、Seller ID 和店铺信息。

## 数据保存位置

当前数据保存在：

`data/db.json`

已自动备份过一次到：

`data/db.backup-before-cleanup.json`

日常备份可以双击：

`backup-data.bat`

备份文件会保存到：

`backups/`

## 注意

当前版本适合作为公司内网试运行版，建议先给少量员工使用，确认字段、流程和导出格式都符合实际业务。

如果后续多人长期同时使用，建议升级到正式数据库，避免 JSON 文件在高并发下成为风险点。
